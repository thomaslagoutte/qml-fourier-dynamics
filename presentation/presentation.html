The PAC Learning Framework

Slide 03

● The Target Arena: Concept Classes

The Probably Approximately Correct (PAC) framework formalizes the efficiency requirements for learning a target family of functions, known as a Concept Class $\mathcal{C}$:

$\mathcal{C} = \{c_\alpha : x \in \mathcal{X} \to y = c_\alpha(x) \in \mathcal{Y}\}_{\alpha \in \mathcal{A}}$

Definition 1. Efficient PAC Learnability

A concept class $\mathcal{C}$ is efficiently PAC learnable if there exists an algorithm $h$ such that for any $c \in \mathcal{C}$, $\epsilon, \delta > 0$, and distribution $\mathcal{D}$:

  • The Guarantee: With probability $\geq 1 - \delta$, the hypothesis $h$ satisfies $|h(x) - c(x)| \leq \epsilon$
  • Sample Complexity: Required classical data pairs scale as $\text{poly}(n, \epsilon^{-1}, \delta^{-1})$
  • Computational Complexity: Total execution runtime scales as $\text{poly}(n, \epsilon^{-1}, \delta^{-1})$

Parametrized Quantum Circuits

Slide 04

Definition 2. Pauli-Encoded PQC & Fourier Representation

A quantum circuit $U: [0,1]^{d} \rightarrow \mathcal{U}(2^n)$ composed of $\text{poly}(n)$ fixed unitaries and $L \in \text{poly}(n)$ parametrized gates generated by Pauli strings $P_l$:

$V_l(\alpha) := e^{i\pi P_l\alpha_{i_l}} \quad \text{for } 1 \leq l \leq L$

Since Pauli spectra are strictly $\{-1, 1\}$, the generated state vector mathematically requires a finite multidimensional Fourier representation:

$|\phi(\alpha)\rangle = U(\alpha) |0\rangle = \sum_{1 \leq k \leq 2^n } \sum_{ l \in [-L,+L]^d} a_{l,k} e^{i\pi \alpha\cdot l} |k\rangle$

● The PQC Function & Expectation Values

Measuring an observable $\hat{O}$ yields a scalar input-output mapping known as a PQC function:

$f(\alpha) = \langle 0 | U^{\dagger}(\alpha) \hat{O} U(\alpha) | 0 \rangle$

For polynomial depth Pauli encodings, the bounded frequency spectrum guarantees a finite, exact evaluation:

$f(\alpha) = \sum_{l\in[-2L,2L]^d} b_l e^{i\pi\alpha \cdot l}$

We call the coefficients $b_l$ the Fourier coefficients. These discretize the continuous parameter evolution.

The PQC Concept Class

Slide 05

Definition 3. The $\mathcal{C}^{PQC}_f$ Concept Class

Let $x \in \{0,1\}^n$ be the bitstring describing the fixed gates, and $\alpha \in [0,1]^{d_n}$ be the unknown parameters. The parametrized state vector is defined as:

$U(x,\alpha) |0\rangle = \sum_{1 \leq k \leq 2^n } \sum_{ l \in [-L,+L]^{d_n}} a_{l,k}(x) e^{i\pi l\cdot \alpha} |k\rangle$

For a given observable $O$ and parameter scaling $d_n = f(n)$, the concept class is the family of expectation mappings over all classical inputs:

$\mathcal{C}^{PQC}_{n,f} := \left\{ c_\alpha : x \to \langle 0 | U(x,\alpha)^{\dagger} O U(x,\alpha) | 0 \rangle \right\}_{\alpha \in [0,1]^{f(n)}}$

Main Theorem Efficient Quantum PAC Learnability

The PQC-based concept class $\mathcal{C}^{PQC}_{\log}$ (restricted to $d = \mathcal{O}(\log n)$) is efficiently quantum PAC learnable. Because the underlying frequency spectrum is bounded, the quantum dynamics can be perfectly extracted and learned from classical data. Both the sample complexity and execution runtime scale strictly polynomially in system size $n$, error tolerance $\epsilon^{-1}$, and failure probability $\delta^{-1}$.

Hardness of the Learning Problem

Slide 06

● $\mathsf{BQP}$-Hardness Embedding

Let $\mathcal{L}$ be a promise $\mathsf{BQP}$ language decided by a polynomial-time quantum circuit $U \in \mathcal{U}(2^n)$. The membership of a string $x$ is determined by the expectation sign of a simple observable $O$:

$\text{sgn}\left(\text{Tr}[O U |x\rangle\langle x| U^\dagger]\right)$

We embed this circuit into the concept class $\mathcal{C}^{PQC}_f$ by injecting parametrized Pauli gates $P_l$ with unknown $\alpha_l$. Crucially, for the specific parameter assignment $\alpha = 0$, the concept evaluates precisely to the underlying $\mathsf{BQP}$-hard problem:

$c_{0}(x) = \langle x | U^\dagger O U | x \rangle$

Main Theorem Classical Intractability Separation

Assuming the widely accepted complexity conjecture that $\mathsf{BQP} \not\subset \mathsf{P/poly}$, no classical algorithm can learn the embedded concept $c_0$. Therefore, the comprehensive concept class $\mathcal{C}^{PQC}_f$ is not classically efficiently PAC learnable. Combined with the quantum efficiency proof, this establishes a strict computational separation.

While this establishes a rigorous mathematical separation, the embedding is an artificial construction. In the subsequent section, we adapt this framework to the compilation of continuous time evolution, bridging the gap to practical, physically motivated learning problems.

Learning Time Evolution

Slide 07

● Towards True Quantum Utility

While the PQC embedding establishes a theoretical separation, demonstrating true quantum utility requires tackling natural physical phenomena. We transition from contrived models to a highly practical application: learning the continuous time evolution of parametrized Hamiltonians.

Definition 4. The Hamiltonian Dynamics Class ($\mathcal{C}^{H}_f$)

Consider a Hamiltonian $H_n(x,\alpha)$ operating on $n$ qubits, defined by inputs $x \in \{0,1\}^{s_n}$ and unknown continuous parameters $\alpha \in [0,1]^{d_n}$. For a fixed evolution time $\tau$, the unitary dynamics are expressed as:

$U_n(x,\alpha) = e^{i\tau H_n(x,\alpha)}$

For a sequence of observables $\{O_n\}_n$ and a parameter scaling $d_n = f(n)$, the corresponding concept class defines the expectation mappings:

$\mathcal{C}^H_{n,f} := \left\{ c_\alpha : x \to \langle 0 | U_n(x,\alpha)^{\dagger} O_n U_n(x,\alpha) | 0 \rangle \right\}_{\alpha \in [0,1]^{f(n)}}$

Physical Target The Learning Objective

The core objective is to efficiently PAC-learn this time evolution strictly from classical input-output pairs. This maps directly to experimental physics: predicting the exact expectation value of an observable after a system evolves under an unknown, naturally occurring physical interaction.

Connecting the Concept Classes

Slide 08

● The Hamiltonian Simulation Bridge

The physical time-evolution class $\mathcal{C}^H_f$ is systematically mapped into the parametrized circuit class $\mathcal{C}^{PQC}_f$ using Hamiltonian Simulation.

By compiling $e^{-i H t}$ into discrete parametrized gates (e.g., via Trotterization), the chosen simulation precision directly dictates:

  • The approximation fidelity between the two classes.
  • The required polynomial depth of the resulting circuit.

Example: A first-order Trotterization translating the Transverse-Field Ising Model (TFIM) into an exact Pauli-encoded PQC structure.

Fundamental Consequence Transfer of PAC Learnability

Because the Hamiltonian evolution compiles perfectly into a polynomial-depth Pauli-encoded circuit, all profound mathematical properties derived for PQCs are strictly preserved. Consequently, the efficient quantum PAC learning algorithm seamlessly adapts to physical systems: the Hamiltonian time-dynamics class $\mathcal{C}^H_{\log}$ inherits provable efficient learnability.

Trotterisation: Compiling Dynamics

Slide 09

1. Truncation Error Bounds

Discretizing the continuous evolution of non-commuting Hamiltonian terms introduces spectral error. To bound this error strictly to $\epsilon$, the required number of time-steps $r$ must scale quadratically with $\tau$:

$\| e^{-iH\tau} - U_r \| \leq \mathcal{O}\left( \frac{\tau^2}{r} \right) \leq \epsilon$

2. Gate Complexity

Each simulated time-step exponentiates $L$ individual Pauli strings. Achieving the precision $\epsilon$ necessitates a total circuit depth $D$ scaling as:

$D \in \mathcal{O}\left( \frac{L \tau^2}{\epsilon} \right)$

Synthesis Link

This discrete Pauli compilation structure structurally maps the Hamiltonian evolution into a Pauli-encoded PQC, guaranteeing the resulting dynamics possess a finite, extractable Fourier spectrum.

The Fourier Extraction Algorithm ($\mathcal{A}$)

Part 1: Subroutine Architecture & Pre-processing

Slide 10

Algorithm $\mathcal{A}$ translates a parametrized quantum circuit into a static circuit that amplitude-encodes its Fourier coefficients. We begin with strict architectural standardisation.

1. Gate Standardisation

Without loss of generality, any Pauli-uploading gate ($X$ or $Y$) is transformed into a $Z$-string using local basis change gates. These local unitaries are subsequently absorbed into the circuit's fixed gate blocks, ensuring all parametrized operations are exclusively of the form $e^{i \alpha_{s_l} \prod Z_k}$.

2. Register Allocation

The expanded state space is partitioned into three distinct registers:

  • Circuit $(c)$: $n$ qubits for the original computational space.
  • Ancillary $(a)$: $1$ qubit dedicated to intermediate parity computation.
  • Frequency $(f)$: Registers $f_s$ tracking the dimension $s$, sized $n_s = 1+\lceil 2\log(\sum s_l)\rceil$.
Total Qubits: $n_T = n + 1 + \sum n_s$

3. The Frequency Basis

The frequency registers are indexed over symmetric negative and positive bounds. We introduce deterministic increment ($V_+$) and decrement ($V_-$) operations enforcing rigid circular boundary conditions: $V_+ |k\rangle = |k+1\rangle$ and $V_+ |k_{max}\rangle = |-k_{max}\rangle$.

The Fourier Extraction Algorithm ($\mathcal{A}$)

Part 2: Gate Transformation Mechanics

Slide 11

Algorithm $\mathcal{A}$ processes the circuit chronologically. Fixed unitaries $U_l^{(c)}$ remain untouched. Each continuous encoding gate is dynamically compiled into a deterministic sequence: $D(b_l) G(s_l) D(b_l)$.

1. The Parity Computer: $D(b_l)$

A cascade of CNOT gates targets the ancillary qubit, controlled strictly by the computational qubits affected by the encoding string $b_l$:

$D(b_l) = \prod_{k \mid b_{l,k}=1} C^{(c_k)} X^{(a)}$

2. The Frequency Shift: $G(s_l)$

Conditioned upon the parity now stored in the ancilla $(a)$, the target frequency register $s_l$ is incremented for even parity or decremented for odd parity:

$G(s_l) = \left( C^{(a)} V_+^{(f_{s_l})} \right) \left( \bar{C}^{(a)} V_-^{(f_{s_l})} \right)$

The continuous $\alpha$-dependence is entirely eradicated, replaced by discrete shifts across the frequency registers.

The Fourier Extraction Algorithm ($\mathcal{A}$)

Part 3: Proof of Correctness by Induction

Slide 12

The proof requires verifying that $\mathcal{A}(U)$ outputs exactly $\sum a_{l,k} |l\rangle^{(f)} |0\rangle^{(a)} |k\rangle^{(c)}$. Steps 1 (Initialization) and 2 (Fixed Gates) operate trivially as identity on $(f)$ and $(a)$. Step 3 (Encoding Gate) forms the inductive core:

Original Circuit Action

The exponential Pauli gate modifies the state $|\phi\rangle$ by applying a phase tied to the parameter $\alpha$ and the substring parity $p(k,b_l) = (-1)^{\sum k_{b_{l}}}$:

$e^{i \alpha_{s_l} \prod Z} |\phi\rangle =$

$\sum_{k,l} a_{l,k} |k\rangle e^{i\alpha\cdot (l + e_{s_l} p(k,b_l))}$

Compiled Action $\mathcal{A}(U)$

Applying $D(b_l)G(s_l)D(b_l)$ mathematically mimics the exact same phase accumulation onto the frequency register $|l\rangle$:

1. $D(b_l):$ $\sum a_{l,k} |l\rangle^{(f)} \left( X^{p} |0\rangle \right)^{(a)} |k\rangle^{(c)}$
2. $G(s_l):$ $\sum a_{l,k} \mathbf{|l + e_{s_l} p\rangle^{(f)}} \left( X^{p} |0\rangle \right)^{(a)} |k\rangle^{(c)}$
3. $D(b_l):$ $\sum a_{l,k} |l + e_{s_l} p\rangle^{(f)} |0\rangle^{(a)} |k\rangle^{(c)}$

The induction concludes: The compiled discrete indices perfectly track the continuous parameters.

Evaluating Expectation Values

Part 4: The Composite State & Post-Selection

Slide 13

1. The Composite State

To extract the observable's Fourier series $f(\alpha) = \langle 0 | U(\alpha)^\dagger P U(\alpha) | 0 \rangle$, we encapsulate the expectation sequence into a single state vector:

$|\phi(\alpha), P\rangle := U(\alpha)^\dagger P U(\alpha) |0\rangle$

2. Compilation & Collapse

Applying $\mathcal{A}$ to this sequence yields a massive superposition. By taking the inner product with $\langle 0|$ on the computational $(c)$ and ancillary $(a)$ registers, the state collapses strictly to the amplitude-encoded coefficients:

$\mathcal{A}(U^\dagger P U) |0\rangle_{all} = \frac{1}{\|b\|_2} \sum_{l \in \mathcal{L}} b_l |l\rangle^{(f)} |0\rangle^{(c,a)} + |\text{trash}\rangle$

Crucial Step: The post-selection success probability is $\sum |b_k|^2$. This isolates the $b_l$ vector, preparing it for explicit extraction.

To extract a specific coefficient $b_l$ from the superposition state, we use a controlled version of the preparation circuit. By applying a target-specific phase shift $V_l^\dagger$ to the frequency register, we isolate the desired index before interfering it via a final Hadamard gate.

Classical Optimization via LASSO

Part 1: The Linearised Learning Problem & Noise Model

Slide 15

1. Linearising the Concept

By extracting the Fourier coefficients $b(x)$, the quantum expectation value is perfectly linearised. The unknown continuous parameters $\alpha$ are absorbed entirely into an unknown weight vector $w(\alpha)$:

$y_t = b(x_t) \cdot w(\alpha) \quad \text{where} \quad w_l(\alpha) := e^{i\pi l\cdot\alpha}$

2. Empirical Matrix & Hardware Noise

The quantum extraction is subject to finite shot noise. Stacking the extracted coefficients forms an empirical feature matrix $\hat{B}$. The error matrix $E$ is strictly bounded by the chosen measurement accuracy $\epsilon_b$:

$E = B - \hat{B} \quad \text{such that} \quad \|E\|_{\infty} \leq \epsilon_b$

3. The Restricted Hypothesis Class ($\mathcal{H}$)

Because $w_l$ are complex phases, the true weight vector has a bounded $\ell_1$-norm ($\|w\|_1 = m$). Our classical learning algorithm searches within the strictly bounded hypothesis class:

$\mathcal{H} = \left\{ h_w : x \to \hat{b}(x)\cdot w \mid \|w\|_1 \leq \Lambda_1 \right\} \quad \text{where } \Lambda_1 = m$

Classical Optimization via LASSO

Part 2: Empirical Risk Minimisation & Sample Complexity

Slide 16

To solve the linear system $Y = \hat{B}w$ under noise, we utilize LASSO regression ($\ell_1$-regularized least squares). This minimizes the Mean Squared Error (MSE) while inherently enforcing sparsity on the exponentially large, but mostly empty, frequency spectrum.

● The Expected Loss Function

The objective is to minimize the generalisation error over the true data distribution $\mathcal{D}$:

$\mathcal{L}_c(h) = \mathbb{E}_{x\sim \mathcal{D}}\left[ |h(x) - c_{\alpha}(x)|^2 \right]$

Statistical Guarantee Sample Complexity via Rademacher Bounds

By bounding the Rademacher complexity of the hypothesis class $\mathcal{H}$ (with $\|w\|_1 = m$ and feature norm $r_\infty = 1$), statistical learning theory guarantees that $\mathcal{L}_c(h) \leq \epsilon$ with probability $1 - \delta$ provided the classical dataset size $T$ satisfies:

$T \geq \frac{16 m^4 \sqrt{ 2 \log\left( \frac{2m}{\delta} \right)}}{\epsilon^2}$

Since $m \in \mathcal{O}(\text{poly}(n))$ for logarithmic depths, the required sample complexity is strictly $\tilde{\Theta}(\text{poly}(n, \epsilon^{-1}, \delta^{-1}))$.

Synthesis & Total Complexity

Part 3: Training, Inference, and Provable Efficiency

Slide 17

Training Phase

To satisfy the regression bounds, the quantum shot noise must be tightly constrained to $\epsilon_b = \frac{0.2 \epsilon}{m}$.

Extracting $m$ frequencies across $T$ samples requires $\mathcal{O}(\epsilon_b^{-2})$ shots per coefficient, bounding the total quantum executions to:

$\text{Runtime}_{train} \in \tilde{\Theta}\left( \frac{m^7}{\epsilon^4} \right)$

Inference Phase

Given a novel, unobserved data point $x_{t'}$, the classical model does not need to be retrained.

We simply extract the corresponding features $\hat{b}(x_{t'})$ using the quantum subroutine and compute the prediction $\hat{y}_{t'} = \hat{w} \cdot \hat{b}(x_{t'})$:

$\text{Runtime}_{infer} \in \tilde{\Theta}\left( \frac{m^3}{\epsilon^2} \right)$

Q.E.D. Proof of Efficient PAC Learnability

Because the frequency count scales as $m = (4L+1)^d$, restricting the parametrization to $d \in \mathcal{O}(\log n)$ guarantees that both the training scaling $\tilde{\Theta}(m^7/\epsilon^4)$ and the inference scaling $\tilde{\Theta}(m^3/\epsilon^2)$ remain strictly bounded by polynomials in system size $n$, error tolerance $\epsilon^{-1}$, and failure probability $\delta^{-1}$.

Consequently, the complete hybrid pipeline efficiently PAC-learns unknown Hamiltonian dynamics from classical data, fulfilling the criteria for a provable exponential quantum-classical separation.

Extension to Hamiltonian Dynamics

Part 4: Absorbing Simulation Error in the PAC Framework

Slide 18

To formally extend the PAC learnability from the circuit class $\mathcal{C}^{PQC}_f$ to the physical class $\mathcal{C}^H_f$, we must account for the deterministic noise introduced by the Hamiltonian simulation compiler.

1. Total Error Decomposition

The total prediction error on the physical system is strictly bounded by the triangle inequality, splitting into the classical statistical learning error and the quantum simulation error ($\epsilon_y$):

$\epsilon_{total} \leq \epsilon_{learning} + \epsilon_y \leq \epsilon$

2. Bound Preservation

To guarantee $\epsilon_y \leq \frac{\epsilon}{2}$, the Trotterization step count $r$ must scale as $\mathcal{O}(\tau^2 L / \epsilon)$. Crucially, because the number of local interactions $L \in \text{poly}(n)$, the compiled circuit depth remains strictly bounded:

$D_{compiled} \in \mathcal{O}\left( \text{poly}(n, \epsilon^{-1}) \right)$

Main Result Learnability of the Physical Target

By absorbing the Trotterization error into the structural noise model, the compiled time-evolution circuit behaves exactly as a PQC with a polynomially bounded frequency spectrum $m$.

Therefore, the Hamiltonian dynamics concept class $\mathcal{C}^H_{\log}$ is efficiently quantum PAC learnable. The total runtime of the classical regression and the required quantum circuit depth both remain bounded by $\tilde{\Theta}(\text{poly}(n, \epsilon^{-1}, \delta^{-1}))$, cementing a rigorous mathematical separation for physical quantum evolution.

High-Dimensional Regimes & Quantum Kernels

Part 1: The Dimensionality Bottleneck & The Kernel Trick

Slide 19

1. The Complexity Explosion

When simulating complex or inhomogeneous physical systems where the parameter count scales polynomially ($d \in \text{poly}(n)$), the frequency spectrum explodes exponentially:

$m = |\mathcal{L}| = (4L+1)^d \implies \text{Explicit LASSO extraction fails.}$

2. The Implicit Kernel Trick

Instead of extracting the exponentially large vector $b(x)$, we pivot to computing the implicit inner product between two feature maps. This defines the exact Quantum Fourier Kernel:

$k(x, x') = \langle b(x) , b(x') \rangle = \sum_{l \in \mathcal{L}} b_l(x)^* b_l(x')$

Kernel Evaluation Circuit (Fig. 8)

By evaluating overlap fidelity directly on the QPU, we bypass the exponential frequency extraction entirely.

High-Dimensional Regimes & Quantum Kernels

Part 2: Kernel Ridge Regression (KRR) Formalism

Slide 20

● Constructing the Empirical Gram Matrix

Given a classical dataset of $T$ samples, we execute the overlap circuit on the QPU pairwise for all inputs to construct the $T \times T$ Gram matrix $\hat{K}$, subject to statistical shot noise:

$\hat{K}_{ij} \approx k(x_i, x_j) = \langle b(x_i), b(x_j) \rangle$

● Tikhonov ($\ell_2$) Regularization

Unlike LASSO which forces $\ell_1$ sparsity, KRR applies an $\ell_2$ penalty to prevent overfitting in the implicitly infinite-dimensional feature space. The optimal weight vector $c$ admits a closed-form analytical solution:

Loss Objective $\min_c \|Y - \hat{K}c\|_2^2 + \lambda c^T \hat{K} c$
Optimal Solution $c = (\hat{K} + \lambda I)^{-1} Y$

New Sample Complexity Paradigm: In this regime, generalisation bounds no longer depend on the explicit spectrum size $m$, but rather on the geometric margin and the Frobenius norm of the Gram matrix.

High-Dimensional Regimes & Quantum Kernels

Part 3: Exponential Concentration & The Sparsity Rescue

Slide 21

1. Exponential Concentration

For highly expressive, high-dimensional PQCs ($d \propto n$), Hilbert space volume grows exponentially. The probability of two random states overlapping vanishes. Consequently, the off-diagonal elements of the Gram matrix exponentially decay:

$k(x_i, x_j) \approx \delta_{ij} + \mathcal{O}(2^{-n})$

2. Generalisation Failure

As $\hat{K}$ collapses to the Identity matrix $I$, the model becomes an exact memorizer. It fits the training data perfectly ($c \approx Y$), but for any unseen inference point $x'$, the kernel evaluates to $0$.

$\hat{y}' = \sum c_i k(x', x_i) \to 0 \implies \text{No Predictive Utility}$

Theorem Restriction The Sparse / Projected Kernel Rescue

PAC efficiency in the high-dimensional regime is rigorously maintained only if the underlying physical dynamics exhibit an intrinsic structural sparsity (i.e., the true non-zero Fourier support remains bounded by $\text{poly}(n)$), or if we apply Localized Projected Kernels that measure overlaps on reduced marginal subsystems to mitigate global Hilbert space dilution.

Quick Start

main.py 
import numpy as np
from quantum_learning_dynamics import Experiment
from quantum_learning_dynamics.hamiltonians import InhomogeneousTFIM
from quantum_learning_dynamics.observables import StaggeredMagnetization

model = InhomogeneousTFIM(num_qubits=4)
observable = StaggeredMagnetization(num_qubits=4)

exp = Experiment(
    model=model,
    observable=observable,
    method="kernel",
    execution_mode="emulator",
    shots=4000,
    tau=1.0,
    r_steps=2,
    kernel_alpha=0.05,
    seed=42
)

result = exp.run(num_train=30, num_test=10)

print(f"Trotter Generalization MSE: {result.mse_trotter:.6f")
print(f"Exact Physics MSE:          {result.mse_exact:.6f")
